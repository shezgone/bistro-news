# BISTRO-News

**뉴스 기반 인과관계 분석 — 거시경제 모델용**

뉴스 기사에서 경제·금융 **이벤트를 추출**하고, 이벤트 간 **선후·인과관계 그래프**를 만들어
**지수(index)**로 압축한 뒤, 거시경제 예측 모델의 입력 신호로 활용하는 것을 목표로 한다.

> BISTRO 거시경제 분석 협업 프로젝트. 본 레포는 기획팀·개발팀 공동 작업 공간이다.

---

## 파이프라인 한눈에

```
뉴스 수집 → 중복제거 → 이벤트 추출(LLM) → 클러스터링 → 인과 그래프 → 지수화 → 거시모델 입력
 (BIGKinds)  (SHA256+   (Step A/B/C/D)              (3단계 필터)  (Track A/B)  (ARIMA/LSTM…)
             BGE-M3)
```

| 단계 | 설명 |
|---|---|
| 1. 수집·중복제거 | 기사 수집 → SHA256(동일) + BGE-M3 임베딩(유사) 클러스터링, `cluster_uuid` 관리 |
| 2. 이벤트 추출 | LLM 4-step: **A** Core → **B** Background → **C** Consequence → **D** Quality Critic |
| 3. 클러스터링 | 추출 이벤트를 임베딩 유사도로 동일 계열끼리 군집화 |
| 4. 인과 그래프 | 시간방향성 강제 → 가중치(선행성×유사도×보도빈도) → LLM+PCMCI 이중검증 |
| 5. 지수화 | Track A(보도강도) / Track B(eigenvector centrality Δ) → 일별 지수 |
| 6. 검증 | Granger 인과검정 + lead-lag 구조 파악 → 역방향으로 스펙 재조정 |

근거 논문: **Tilly & Livan (UCL), 2021**, *Macroeconomic forecasting with statistically validated knowledge graphs*, [arXiv:2104.10457](https://arxiv.org/abs/2104.10457).

## 기획팀 ↔ 개발팀 협업 가이드

- **프롬프트 개선** → `prompts/` 에서 작업. 단계(A/B/C/D)별로 버전 관리하고, 변경 이유를 커밋 메시지에 남긴다.
- **출력 스키마 변경** → `schemas/event_schema.json` 이 추출 결과의 **계약(contract)**. 필드·enum 추가/변경 시 반드시 이 파일을 먼저 합의 후 수정한다.
- **설계·방법론 논의** → `CLAUDE.md` 가 권위 스펙. 큰 결정은 여기 반영하고, 회의·중간 요약은 `docs/` 에 날짜 붙여 보관한다.
- **브랜치 전략** → `main` 직접 푸시 대신 기능 브랜치 + PR 권장. 리뷰로 스키마 호환성 확인.

## 데이터 수집 & 저작권 ⚠️

한국 기사는 대부분 **저작권 보호 대상**이다(단순 사실전달만 예외, 저작권법 §7). 임의 크롤링 대신 **라이선스된 소스**를 사용한다.

| 경로 | 본문 | 비용 | 비고 |
|---|---|---|---|
| **BIGKinds**(한국언론진흥재단) | ○ | 유상(정황) | 공공·연구기관용 정식 경로. 1순위 |
| 연합뉴스 직접 데이터계약 | ○ | 유상 | 기관 직접계약 용이 |
| 공공데이터포털 KPF 데이터셋 | ✕(메타) | 무료 | 보조용 |
| GDELT | △ | 무료 | 글로벌 이슈·baseline·교차검증 |

- BIGKinds 오픈API는 최근 **일반 이용 유상화** 정황 → 정확한 조건은 KPF 뉴스빅데이터팀(02-2001-7574) 확인 필요.
- **copyright-by-design**: 기사 본문은 해시·임베딩 후 폐기하고, **추출된 구조화 사실(이벤트)만 보존**한다. 외부 LLM 전송 시 라이선스 범위·역외전송을 별도 확인한다.
